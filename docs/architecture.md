## 组件说明

### 1. Supervisor 进程管理器

作为容器的主进程（PID 1），负责管理：
- DBus系统总线（微信桌面通知依赖）
- TigerVNC服务器（提供图形界面）
- 微信支付回调服务（Python应用）
- 可选：微信客户端自动启动

### 2. TigerVNC 服务器

- **显示编号**: :1
- **端口**: 5901
- **功能**: 提供远程桌面访问
- **特点**: 支持多客户端连接，性能优异

### 3. noVNC + Websockify

- **端口**: 6080
- **功能**: 将VNC协议转换为WebSocket，支持浏览器直接访问
- **优点**: 无需安装VNC客户端，随时随地浏览器访问

### 4. XFCE 桌面环境

- **选择理由**: 轻量级，资源占用低，适合远程访问
- **包含组件**: 桌面、面板、终端、文件管理器
- **中文支持**: 预装中文字体和fcitx输入法

### 5. 微信Linux版

- **安装路径**: `/opt/wechat/wechat`
- **启动命令**: `start-wechat` (封装脚本)
- **数据持久化**: `/root/.config/wechat`

### 6. 微信支付回调服务

- **端口**: 8888（API）、8889（管理）
- **监听来源**: 
  - `biz_message_0.db`（提前识别支付消息）
  - `session.db`（后备检测）
- **去重机制**: 跨来源30秒时间窗口去重

## 网络架构

### 端口映射

```
宿主机                      容器
─────────────────────────────────────────────────
5901  ───────────────────►  5901 (VNC原生)
6080  ───────────────────►  6080 (noVNC)
8888  ───────────────────►  8888 (支付API)
8889  ───────────────────►  8889 (管理API)
```

### 内部网络

```
Docker Bridge Network (wechat-vnc-net)
    │
    ├──── wechat-vnc (172.28.0.2)
    │       ├── VNC :5901
    │       ├── noVNC :6080
    │       ├── API :8888/:8889
    │
    └──── nginx (172.28.0.3, optional)
            ├── HTTP :80
            └── HTTPS :443
```

## 存储架构

### 持久化卷

```
宿主机路径                      容器路径                     用途
─────────────────────────────────────────────────────────────────────────
./data                     →  /app/data                  应用数据
./data/wechat              →  /root/.config/wechat       微信配置和登录状态
./data/fcitx               →  /root/.config/fcitx        输入法配置
./data/decrypt             →  /root/wechat-decrypt        解密数据库
./data/vnc                 →  /root/.vnc                  VNC密码和配置
./logs                     →  /var/log/supervisor        日志文件
./config.yaml              →  /app/config.yaml            服务配置
```

## 故障恢复机制

### 自动重启策略

| 组件 | 重启策略 | 最大重试 |
|------|----------|----------|
| DBus | always | 无限 |
| TigerVNC | always | 无限 |
| 支付回调 | always | 无限 |
| 微信客户端 | on-failure | 3次 |

## 扩展性设计

### 多实例部署

```yaml
# docker-compose.scale.yml
services:
  wechat-vnc-1:
    extends:
      file: docker-compose.vnc.yml
      service: wechat-vnc
    ports:
      - "5901:5901"
      - "6080:6080"
      - "8888:8888"
    volumes:
      - ./data/instance1:/app/data

  wechat-vnc-2:
    extends:
      file: docker-compose.vnc.yml
      service: wechat-vnc
    ports:
      - "5902:5901"
      - "6081:6080"
      - "8889:8888"
    volumes:
      - ./data/instance2:/app/data
```

## 性能指标

### 资源占用参考

| 组件 | CPU | 内存 | 说明 |
|------|-----|------|------|
| 基础系统 | 0.1核 | 200MB | Ubuntu + XFCE |
| TigerVNC | 0.1核 | 50MB | 空闲时 |
| 微信 | 0.5核 | 300MB | 运行时 |
| 回调服务 | 0.1核 | 100MB | Python应用 |
| **总计** | **0.8核** | **650MB** | 推荐预留2GB |

### 优化建议

1. **降低分辨率**：VNC_RESOLUTION=1024x768 可显著减少带宽和CPU
2. **限制帧率**：VNC客户端可设置"低质量模式"
3. **使用JPEG压缩**：TigerVNC默认启用
